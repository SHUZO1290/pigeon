# Pigeon CRM

Модульная веб-платформа для учёта клиентов и управления точками обслуживания с гибкой ролевой моделью.

## Стек

- **Frontend:** Vue 3 (Composition API), Vite, Pinia, Vue Router, Bootstrap 5, Chart.js
- **Backend:** Python 3, FastAPI, SQLAlchemy, SQLite, JWT (python-jose), Passlib
- **API:** REST, Swagger

## Роли и иерархия

1. **Владелец (Owner)**
   - Создаёт управляющих
   - Создаёт точки и назначает в них управляющих
   - Может входить под учётной записью любого управляющего

2. **Управляющий (Manager)**
   - Создаёт кассиров
   - Управляет своими точками: строит гибкие таблицы  с типами данных и выпадающими списками
   - Привязывает кассиров к точкам
   - Может удалять таблицы и кассиров
   - Может входить под учётной записью своих кассиров

3. **Кассир (Cashier)**
   - Видит только свою кассу (список таблиц привязанных точек)
   - Добавляет, редактирует и удаляет строки в таблицах
   - Быстрое добавление клиента через боковую панель

## Модули системы

| Модуль | Описание |
|--------|----------|
| Аутентификация | JWT-токены, ролевой доступ, имперсонация |
| Дашборд | Сводная аналитика для каждой роли |
| Управляющие | CRUD, вход под управляющим |
| Кассиры | CRUD, вход под кассиром |
| Точки | Привязка управляющих и кассиров |
| Таблицы | Конструктор колонок (текст, число, дата, выпадающий список) |
| Касса | Редактирование строк динамических таблиц |

## Быстрый старт

### 1. Клонируйте репозиторий
### 2. Запуск бэкенда
```
cd pigeon-backend
python -m venv .venv
# Linux:
source .venv/bin/activate
# Windows:
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```
Сервер запустится на http://localhost:8000.
Документация: http://localhost:8000/docs
### 3. Запуск фронтенда
```
cd pigeon-frontend
npm install
npm run dev
```
Приложение откроется на http://localhost:3000.
Прокси настроен автоматически (Vite перенаправляет запросы к API на бэкенд).
### 4. Вход в систему
По умолчанию создаётся владелец:
- Логин: owner
- Пароль: 123456
Остальные пользователи создаются через интерфейс согласно ролям.

## Реализация
- Модульная архитектура: каждый раздел меню – независимый роутер FastAPI и отдельный модуль фронтенда.
- Ролевая модель: трёхуровневая иерархия с имперсонацией.
- Динамические таблицы: хранение метаданных колонок и данных строк в JSON, без жёсткой схемы.
- Безопасность: JWT-токены, хеширование паролей, заголовки CORS.
- Swagger-документация: все эндпоинты доступны для тестирования прямо в браузере.
